# PulsePass Design System Rules

- **Framework**: Custom UI Components styled with Vanilla CSS (CSS Modules). Do NOT use Shadcn UI or Tailwind CSS.
- **Tokens**: Use the pre-existing design tokens defined in `theme-tokens.css` (e.g., `--color-primary`, `--font-headings-h1-regular-font-size`) for all styling.
- **Typography**: Inter, Satoshi, General Sans.
- **Accessibility**: Must meet WCAG 2.2 AA specifications. Provide Keyboard Navigation, Focus States, Screen Reader Support, Semantic HTML, and Contrast Compliance.
- **Core Principles**: Clarity, Trust, Speed, Delight.
- **Components Target**: Standardize Buttons, Forms, Inputs, Cards, Tables, Charts, Dialogs, Drawers, Tabs through custom component implementations.
